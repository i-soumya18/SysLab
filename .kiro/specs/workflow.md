# 🧭 The Golden User Journey

## STEP 0 — Landing & Value Clarity (30 seconds)

User lands on homepage.

They immediately see:

> "Build a system. Scale it. Watch it break. Fix it."

**CTA:**
- "Start Designing"
- "Try a Demo System"

No heavy signup wall.

---

## STEP 1 — Onboarding (2 minutes)

After signup / guest mode:

User chooses:
- 🔹 "Learn System Design"
- 🔹 "Practice Interviews"
- 🔹 "Free Play"

👉 This determines what canvas opens.

---

## STEP 2 — Canvas Opens (First Wow Moment)

User sees:
- Empty canvas
- Component library on the left
- Metrics panel on the right
- Traffic slider at the top

System gently nudges:

> "Drag a Load Balancer to start."

---

## STEP 3 — Build the First System (Hands-On)

User drags:
- Client
- Load Balancer
- Service
- Database

Wires them together visually.

System auto-validates connections.

💡 Tooltips explain what each component does.

---

## STEP 4 — Run Simulation (The "Oh Sh*t" Moment)

User clicks **Run**.

- Traffic = 100 users → system stable
- Traffic = 10K → DB latency rises
- Traffic = 100K → errors appear
- Traffic = 1M → system collapses

🔥 Bottlenecks glow red.

**This is where learning actually happens.**

---

## STEP 5 — Observe Metrics & Cost

User checks:
- Latency graphs
- Error spikes
- Queue depth
- 💸 Monthly cost exploding

System explains:

> "Database is saturated due to single-node writes."

---

## STEP 6 — Fix the System (Core Learning Loop)

User:
- Adds cache
- Adds read replicas
- Introduces queue
- Scales service horizontally

Re-runs simulation.

🟢 System stabilizes at higher scale.

👉 **This loop = learning engine**

```
Build → Scale → Break → Observe → Fix → Repeat
```

---

## STEP 7 — Inject Failures (Advanced Intuition)

User toggles:
- DB node failure
- Network latency spike
- Cache eviction storm

User sees:
- Partial outages
- Cascading failures
- Recovery patterns

**This teaches real-world resilience.**

---

## STEP 8 — Scenario Completion / Interview Feedback

**If in guided mode:**
- System evaluates design
- Highlights tradeoffs
- Shows better alternatives

**If in interview mode:**
- Timer stops
- Feedback summary
- Strengths / weaknesses

---

## STEP 9 — Save, Share, Reflect

User:
- Saves design
- Shares link
- Compares versions
- Returns later

**This builds habit + retention.**

---

## 🔁 Repeat Usage Loop (Why Users Come Back)

- New scenarios
- Higher scale challenges
- Interview practice
- Cost optimization games
- Competitive leaderboards

---

## 🎯 What Success Looks Like (End State)

**From a user's POV:**

> "I understand system design now — I don't memorize answers."

**From a product POV:**
- Long session times
- Repeat usage
- Strong word-of-mouth
- Clear upgrade path

---

## 🔥 TL;DR

### End Deliverable

A **System Design Flight Simulator** that turns abstract concepts into lived experience.

### User Workflow

```
Drag → Simulate → Break → Observe → Fix → Learn
```
